# ProductTrapper Architecture

ProductTrapper is a static-friendly Astro site deployed to Cloudflare Workers. It fetches Product Hunt featured posts once per day, normalizes them into JSON, and renders an interactive single-page UI.

## High-level flow

```
Product Hunt GraphQL API v2
        │
        ▼
scripts/fetch-posts.js  (prebuild, Node.js)
        │
        ▼
src/generated/posts.json
        │
        ├──► Astro build (server import)  ──► dist/
        │
        └──► public/posts.json            ──► served to browser
```

## Build-time data pipeline

1. `npm run build` triggers `prebuild` via npm hook.
2. `scripts/prebuild.js` calls `scripts/fetch-posts.js` unless `SKIP_PREBUILD=true`.
3. `scripts/fetch-posts.js` pages through `posts(featured: true, postedAfter: "2026-01-01T00:00:00Z")`, handles rate limits, normalizes records, and writes:
   - `src/generated/posts.json` — consumed by Astro pages
   - `public/posts.json` — consumed by the browser React island
4. `--incremental` fetches only posts newer than the latest cached `featuredAt`.

## Runtime

- `src/pages/index.astro` is prerendered to a static page.
- It loads topics/metadata at build time via `src/lib/posts.ts` and hydrates `PostBrowser` with `client:load`.
- `PostBrowser` fetches `public/posts.json` on the client, filters by topic, and switches between list/ranking views.
- URL query state (`?view=ranking&topic=ai`) is synced via `history.replaceState`.

## Directory structure

```
src/
├── components/
│   ├── astro/          # Static Astro components (Footer)
│   └── react/          # Interactive islands (PostBrowser, PostGrid, RankingList, TopicTabs, ViewToggle)
├── layouts/
│   └── BaseLayout.astro
├── lib/
│   ├── constants.ts    # SITE_URL / SITE_NAME
│   ├── data-client.ts  # Browser fetch helper for public/posts.json

├── messages/
│   └── ja.json         # UI labels
├── pages/
│   └── index.astro
├── generated/
│   └── posts.json      # Generated by prebuild (gitignored)
└── styles/
    └── globals.css
```

## Components responsibilities

| Component | Type | Responsibility |
|-----------|------|----------------|
| `BaseLayout` | Astro | HTML boilerplate, SEO, global styles, footer injection |
| `Footer` | Astro | Product Hunt attribution and last-updated date |
| `PostBrowser` | React | Fetch data, manage view/topic state, render list or ranking |
| `ViewToggle` | React | Switch between `list` and `ranking` |
| `TopicTabs` | React | Horizontal scrollable topic filter tabs |
| `PostGrid` | React | Card grid layout with thumbnails and metadata |
| `RankingList` | React | Numbered ranking list with vote counts |

## Deployment

- Astro `output: 'server'` with `@astrojs/cloudflare` adapter.
- `wrangler.toml` defines the Worker name (`producttrapper`) and compatibility flags.
- GitHub Actions handle scheduled daily updates and push-based deploys.

## Security notes

- `PH_DEVELOPER_TOKEN` is only read by Node.js prebuild scripts from `.env.local` / GitHub Secrets.
- The token is not referenced in Astro/React code, so it never reaches the client or Worker runtime.
